Day 7 · Behind the Scenes

🔐 The OAuth Problem Nobody Talks About

Post at 9 AM IST
Day 7
LinkedIn Post
Here's the engineering challenge I didn't expect. Zomato's OAuth only works with: → Claude.ai (whitelisted) → VSCode.dev (whitelisted) → ChatGPT (whitelisted) Gemini CLI? Blocked. Claude Code CLI? Blocked. So I built a bridge. A local proxy server that: 1. Uses Claude Desktop's whitelisted OAuth to get the token 2. Stores it at ~/.zomato-token 3. Forwards it to any CLI tool on request Result: Every tool gets authenticated. Zero workarounds for the user. The best solutions solve the problem invisibly. Users just type a command. The complexity disappears. 💡
#OAuth #Engineering #MCP #ProblemSolving
